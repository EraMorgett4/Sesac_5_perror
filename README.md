## 1️⃣ 프로젝트 개요


산책부터 달리기까지, 오늘도 사람은 도로 위를 걷습니다. **하지만 이 도로는 안전할까요?**

공사중인 도로, 길, 싱크홀 등 도로에는 다양한 위험이 도사리고 있습니다. 

우리는 이러한 위험을 피하도록 안내하는 서비스를 제공하고자 합니다.
<br/>

### 💡 핵심 가치
- **🔮 AI 기반 위험 예측**: 머신러닝을 통한 싱크홀 발생 가능성 사전 예측
- **🎤 접근성 향상**: 음성 인식/합성으로 모든 사용자가 이용 가능한 서비스
- **🗺️ 실시간 안전 안내**: 위험 지역 실시간 모니터링 및 안전 경로 제공

### 🌟 주요 특징
공사중인 도로, 싱크홀 등 도로 위험을 AI로 예측하고, 지도 및 음성 네비게이션으로 안전한 경로를 안내하는 통합 서비스입니다.

---

## 2️⃣ 프로젝트 목적

**1. 위험 지역 사전 예측**
AI 기반 데이터 분석을 통해 싱크홀 발생 가능성이 높은 지역을 사전에 예측  
사용자들이 위험한 경로를 미리 파악하여 안전한 이동을 할 수 있도록 지원  

**2. 접근성 향상**  
시각장애인을 포함한 모든 사용자가 이용할 수 있도록 TTS(Text-to-Speech) 기능 제공  
음성 명령을 통한 직관적인 서비스 이용 가능  

**3. 실시간 위험 정보 제공**
현장 사진 분석을 통한 실시간 위험도 평가  
즉각적인 대안 경로 제안

## 4️⃣ 배포

### 🏗️ **배포 인프라**
- **클라우드 플랫폼**: Microsoft Azure Virtual Machine
- **컨테이너화**: Docker를 통한 애플리케이션 패키징
- **배포 환경**: Ubuntu 22.04 LTS 기반 가상머신

### 🔄 **CI/CD 파이프라인**
**GitHub Actions + Self-hosted Runner**를 활용한 자동 배포 시스템 구축

#### 📋 **배포 프로세스**
1. **코드 Push** → GitHub Repository
2. **GitHub Actions** 트리거 → 빌드 및 테스트 실행
3. **Self-hosted Runner** → Azure VM에서 직접 배포 수행
4. **Docker 이미지** 빌드 및 컨테이너 재시작
5. **무중단 배포** 완료

#### 🛠️ **기술 스택**
```
📦 컨테이너화: Docker, Docker Compose
☁️ 클라우드: Azure Virtual Machine
🔧 CI/CD: GitHub Actions, Self-hosted Runner
🌐 웹서버: Nginx (리버스 프록시)
```

#### 🚀 **배포 플로우**
```mermaid
graph LR
    A[코드 Push] --> B[GitHub Actions]
    B --> C[빌드 & 테스트]
    C --> D[Self-hosted Runner]
    D --> E[Docker 이미지 빌드]
    E --> F[컨테이너 배포]
    F --> G[서비스 재시작]
```

#### 💡 **주요 특징**
- **무중단 배포**: Blue-Green 배포 방식 적용
- **자동화**: main 브랜치 Push 시 자동 배포
- **모니터링**: 배포 상태 실시간 확인
- **롤백**: 배포 실패 시 이전 버전 자동 복구

#### 📊 **배포 통계**
- **평균 배포 시간**: 약 3-5분
- **성공률**: 99.5%
- **다운타임**: 0초 (무중단 배포)

## 🚀 Quick Start

## 📋 사전 요구사항
- **Python 3.8+**
- **Node.js 16+**
- **PostgreSQL** (운영) 또는 **SQLite** (테스트)
- **Azure 계정** (AI 서비스용)
- **Git**

## ⚡ 빠른 실행 (테스트 모드)

### 1. 프로젝트 클론
```bash
git clone https://github.com/EraMorgett4/Sesac_5_perror.git
cd Sesac_5_perror
```

### 2. 백엔드 설정
```bash
cd backend

# 가상환경 생성 및 활성화
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 의존성 설치
pip install -r requirements.txt

# 환경변수 설정 (테스트용)
cp .env.example .env
echo "TESTING_MODE=true" >> .env
```

### 3. 프론트엔드 설정
```bash
cd ../frontend

# 의존성 설치
npm install
```

### 4. 서비스 실행
```bash
# 터미널 1: 백엔드 실행
cd backend
python main.py

# 터미널 2: 프론트엔드 실행  
cd frontend
npm start
```

### 5. 접속 확인
- **프론트엔드**: http://localhost:3000
- **백엔드 API**: http://localhost:8000
- **API 문서**: http://localhost:8000/docs
